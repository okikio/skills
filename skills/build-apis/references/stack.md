# API stack

Inspect Hono core, runtime adapters, middleware, RPC/client, validators, and
OpenAPI packages. Runtime portability does not imply adapter parity. Inspect
Better Auth server/client packages, database adapters, plugins, sessions,
cookies, generated schema requirements, and framework integration;
authentication does not imply authorization. Keep Zod v4 as the application
schema source where selected, and use Standard Schema for validator-neutral
boundaries.
