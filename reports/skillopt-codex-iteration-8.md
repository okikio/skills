# Codex SkillOpt-style iteration 8

## Scope

A fresh independent optimizer reviewed the rebuilt roots, references, routing,
and non-frozen cases. This iteration applied only bounded changes that reduced
framework bias, retrieval cost, or ambiguity. It did not inspect frozen prompts
while generating edits.

## Accepted changes

- Made `build-sites` and `build-web-apps` ownership framework-neutral, with
  Astro, TanStack, Solid, Better Auth, and LogTape instructions loaded only when
  installed, selected, or under review.
- Added explicit ecosystem materiality, read-only research, stopping conditions,
  and unresolved-evidence reporting.
- Classified CLI public terms as early controls, source-bearing configuration,
  or domain inputs before applying the full configuration trace.
- Made LogTape routing conditional on LogTape being the verified output owner.
- Added exact Astro `client:only="react"` correction guidance.
- Added focused form/mutation and table/virtualization references.
- Added held-out form-race, data-view, incidental-dependency, non-Astro,
  non-Solid, and content-site versus stateful-app cases.
- Removed the detached trainable copy so the optimizer edits the exact
  `SKILL.md` installed and scored.
- Pinned uploaded evidence by digest and validated claim-level source paths.
- Required child web skills to consume the shared surface map instead of
  repeating repository-wide discovery.

## Rejected changes

- Activating the shared `build-web` skill for every clear site or application
  task; the focused skill is sufficient unless cross-surface ownership is needed.
- Deep ecosystem research for every incidental import.
- Forcing Astro, TanStack, Solid, Better Auth, or LogTape into repositories with
  different established owners.
- Adding browser-extension instructions without an implementation evidence set.
- Moving the complete guidebooks into always-loaded root skill text.

## Result

The optimization corpus now gives each rebuilt skill at least two train or
seen-validation cases and held-out coverage appropriate to its scope. This is a
retrieval and behavioral-contract improvement, not a numeric pass-rate claim.
